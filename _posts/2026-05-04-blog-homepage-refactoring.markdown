> "好的设计不是让用户去找内容，而是让内容主动找到用户。"

作为一名小学数学教师，搭建这个博客的初衷很简单：**记录教学心得，分享实用工具**。但在实际使用过程中，我发现原有的「独立教学工具页面」存在一些体验上的不足。经过一番思考与实践，我决定对博客首页进行重构——**将教学工具从独立页面迁移至首页，以分类卡片的形式直接展示**。

[跳过铺垫，直击核心思考](#core)

这次调整并非一时兴起，而是基于实际使用中的痛点分析。下面我将从**调整思路**、**实际收益**、**维护方式**和**后续改进**四个维度，完整记录这次重构的过程与心得。

## 一、为什么做这次调整

### 原有模式的问题

在调整之前，我的博客采用的是经典的「多页面分离」架构：

| 页面 | 内容 | 访问路径 |
|------|------|----------|
| 首页 | 文章列表 | `/` |
| 关于我 | 个人介绍 | `/about/` |
| 随笔归档 | 全部文章 | `/archive/` |
| **教学工具** | **工具列表** | **`/tools/`** |

看起来结构清晰，但实际使用中暴露出几个问题：

1. **入口隐藏过深**：教学工具藏在导航栏的一个小链接里，学生和家长很难第一时间发现
2. **认知负担大**：需要先理解「首页是文章」「工具在另一个页面」的逻辑
3. **移动端体验差**：手机上要多次点击才能到达工具页面

### 核心目标

我想要实现的效果其实很朴素：

> **打开首页就能看到最新随笔 + 所有教学工具，一屏之内完成核心操作。**

## 二、最终采用的方案

### 首页新布局

重构后的首页采用**上下双区结构**：

```
┌─────────────────────────────┐
│        头部横幅图片           │
├─────────────────────────────┤
│                             │
│   📝 最新随笔（仅显示1篇）    │  ← 顶部随笔区
│   《相等城堡听课 & 代数讲座》  │
│                             │
├ ─ ─ ─ 教学工具 ─ ─ ─ ─ ─ ─ ┤  ← 分隔线
│                             │
│   数学练习（3个工具）         │
│   ┌───┐ ┌───┐ ┌───┐       │
│   │加法│ │计算│ │成绩│       │  ← 教学工具区
│   └───┘ └───┘ └───┘       │     （3列卡片布局）
│                             │
│   几何图形（6个工具）         │
│   ┌───┐ ┌───┐ ┌───┐       │
│   │圆锥│ │圆柱│ │组合│       │
│   └───┘ └───┘ └───┘       │
│   ...                        │
│                             │
├─────────────────────────────┤
│  FEATURED TAGS / ABOUT ME   │  ← 右侧边栏（不变）
│  LINK                       │
└─────────────────────────────┘
```

### 关键技术决策

#### 决策一：随笔只显示最新1篇

原来的首页会显示所有文章的分页列表。但考虑到：

* 我的写作频率不高（平均每月1-2篇）
* 旧随笔的价值在于归档检索，而非首页展示
* 首页应该聚焦于**最新动态**

因此改为**只展示最新1篇随笔**，其余通过「随笔」归档页访问。

#### 决策二：工具自动扫描 + 智能分类

这是本次重构的核心创新点。我设计了**双层机制**：

| 层级 | 来源 | 展示信息 |
|------|------|----------|
| **已配置层** | `_data/tools.yml` | 自定义 emoji、标题、描述 |
| **自动发现层** | 扫描 `tools/` 目录 | 默认图标🛠️ + 文件名作为标题 |

这意味着：
* 已配置的工具 → 显示精美信息（emoji + 中文标题 + 详细描述）
* 新放入的工具 → 自动出现（无需任何配置）
* 后续想美化 → 只需编辑 `tools.yml`

#### 决策三：按学科分类分组

21个教学工具如果平铺显示会非常混乱。我根据文件名关键词，将它们分为6大类：

| 分类 | 数量 | 包含工具示例 |
|------|------|-------------|
| 数学练习 | 3个 | 加法练习、趣味计算器、成绩管理 |
| 几何图形 | 6个 | 圆锥体积、圆柱表面积、组合图形 |
| 比例问题 | 5个 | 比例教具、比例尺行程、按比例分配 |
| 数的运算 | 2个 | 四舍五入、大数读写改写 |
| 时间日期 | 2个 | 星期计算、年月日计算 |
| 其他 | 3个 | 古诗大战僵尸、台秤演示、人民币兑换 |

每个分类用**左侧蓝色竖线标题**区分，视觉层次清晰。

## 三、这次调整带来的好处

### 对教师（我自己）

**1. 维护成本大幅降低**

以前添加一个新工具需要4步：
1. 把 HTML 文件放入 `tools/`
2. 编辑 `tools.html` 手动添加卡片代码
3. 编辑 `_data/tools.yml` 配置信息
4. 提交部署

现在只需要**1步**：把 HTML 文件放入 `tools/`，刷新首页即可看到。

如果后续想补充图标和描述，随时可以编辑 `tools.yml`，但不影响工具的正常显示。

**2. 内容管理更直观**

首页现在就是整个博客的**全景图**：
* 最新写了什么 → 看顶部随笔区
* 有哪些工具可用 → 往下滚动即可
* 不需要在多个页面之间跳转

### 对学生和家长

**1. 降低使用门槛**

以前的使用路径：`首页 → 找到导航栏 → 点击"教学工具" → 浏览工具列表`

现在的使用路径：`首页 → 向下滚动 → 直接看到所有工具`

**2. 分类查找更高效**

工具按学科分类后，学生可以快速定位需要的类型：
* 今天学了圆柱 → 直接看「几何图形」分类
* 要练口算 → 直接看「数学练习」分类

**3. 移动端友好**

3列布局在桌面端展示丰富，在平板端自动变为2列，在手机端变为单列，适配各种设备。

## 四、后期如何维护

### 日常维护流程

```
添加新工具
    │
    ▼
把 HTML 文件放入 tools/ 文件夹
    │
    ├── 方案A：直接发布（推荐）
    │   工具会自动出现在首页对应分类中
    │   显示默认图标🛠️ + 文件名
    │
    └── 方案B：配置后再发布
        编辑 _data/tools.yml
        添加 emoji、title、description
        发布后显示自定义信息
    
提交到 GitHub → 自动部署 → 生效
```

### 推荐的工作习惯

1. **先用方案A快速上线**：先把工具放进去，确认功能正常
2. **有空再完善配置**：编辑 `tools.yml` 补充中文标题和描述
3. **定期整理分类规则**：如果有新类型的工具，更新首页的分类判断逻辑

### 配置文件示例

```yaml
# _data/tools.yml
- filename: Addition within 10.html
  emoji: ➕                    # 选择合适的图标
  title: 10以内加法练习          # 小学生能看懂的中文标题
  description: 帮助一年级学生练习10以内加法运算，通过趣味互动提高口算能力。
                              # 1-2行说明用途
```

## 五、后续改进思路

虽然当前方案已经能满足基本需求，但我还有一些想法，留待未来逐步实现：

### 短期改进（1个月内）

**1. 工具使用统计**

在每个工具卡片上增加「使用次数」显示，帮助我了解哪些工具更受欢迎：

```html
<div class="tool-card">
    <div class="tool-card-header">
        <span class="tool-icon">➕</span>
        <a href="/tools/Addition within 10.html" class="tool-title">10以内加法</a>
    </div>
    <p class="tool-description">帮助一年级学生...</p>
    <div class="tool-meta">
        <span class="tool-uses">已使用 128 次</span>  <!-- 新增 -->
    </div>
    <a href="..." class="tool-btn">打开工具</a>
</div>
```

**2. 收藏功能**

允许学生「收藏」常用工具，收藏后的工具出现在首页顶部或侧边栏的「我的工具箱」区域。

**3. 搜索过滤**

在工具区域上方增加搜索框和分类筛选器：

```html
<div class="tools-filter">
    <input type="text" placeholder="搜索工具..." />
    <div class="filter-tags">
        <span class="active">全部</span>
        <span>数学练习</span>
        <span>几何图形</span>
        ...
    </div>
</div>
```

### 中期改进（2-3个月）

**4. 工具评分与反馈**

每个工具下方增加简单的评分按钮（⭐⭐⭐⭐⭐），收集学生的使用反馈，帮助我持续改进工具质量。

**5. 工具版本管理**

当某个工具有更新时，在卡片上显示「NEW」标签或版本号，让学生知道有新内容可以尝试。

**6. 按年级推荐**

根据工具的适用年级（如一年级、二年级、三年级），在首页增加「年级筛选」选项，让不同年级的学生快速找到适合自己的工具。

### 长期愿景（半年以上）

**7. 学生账号系统**

为学生创建简单账号，记录每个人的使用历史、收藏列表和学习进度，提供个性化的工具推荐。

**8. 工具市场**

开放接口，允许其他教师上传自己制作的教学工具，形成一个**小学数学教学工具共享社区**。

**9. 数据驱动的内容创作**

通过分析工具使用数据和学生反馈，了解哪些知识点学生掌握得不好，从而指导我后续的随笔写作和工具开发方向。

## 六、写在最后

这次首页重构看似只是一个布局调整，但背后反映的是我对**「教师博客定位」**的重新思考。

教师博客不应该是一个自我展示的平台，而应该是一个**真正服务于教学的工具**。每一次点击、每一个交互，都应该围绕「如何帮助学生更好地学习数学」来设计。

从独立页面到首页集成，从手动配置到自动扫描，从平铺展示到分类组织——这些改变的背后，是我希望让技术**隐形化**的想法：

> **最好的技术，是你感觉不到它的存在，但它一直在帮你解决问题。**

接下来，我会继续在这个方向上探索，也欢迎各位同行提出宝贵的意见和建议。
